# Remote Distillation Training (720×320 pipeline)

Three stages, run in order: **teacher_gen → warmup → self-forcing**.

Teacher checkpoint: [Shirk6/dreamdojo-piper-insert-mouse-battery-720-320-10fps-40k](https://huggingface.co/Shirk6/dreamdojo-piper-insert-mouse-battery-720-320-10fps-40k)

## Directory layout (after setup)

```
dreamdojo-distill/
├── .venv/
├── cosmos_predict2/                                # source
├── local_jobs/                                     # launch scripts
├── scripts/                                        # helpers
├── docs/
├── datasets/
│   ├── cr1_empty_string_text_embeddings.pt         # symlink → HF cache, ~200 MB
│   ├── piper_insert_mouse_battery_lerobot/         # raw lerobot source (4 GB)
│   └── piper_720_320_warmup_regenerated_4step/     # generated by stage 1 (~53 GB)
│       ├── actions/{0..9999}.json
│       ├── images/{0..9999}.png
│       ├── videos/{0..9999}.mp4
│       └── latents/{0..9999}.pt                    # dict[int → (16,4,40,90)]
├── checkpoints/
│   └── dreamdojo-piper-insert-mouse-battery-720-320-10fps-40k/     # 22 GB teacher
│       ├── model/{__0..7_0.distcp, .metadata}
│       └── optim/{__0..7_0.distcp, .metadata}
└── dreamdojo_logs/                                 # training outputs (warmup + SF)
```

Requirements: 8 × 80 GB GPU (H200/A100/A800/H100), ≥ 700 GB free disk, CUDA 12.8 driver.

## Step 1 — clone

```bash
git clone https://github.com/Cocoyawn/dreamdojo-distill.git
cd dreamdojo-distill
```

## Step 2 — install python dependencies

```bash
bash install.sh
```

Verify:

```bash
.venv/bin/python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
# expected: 2.7.0+cu128 True
```

## Step 3 — HuggingFace authentication

```bash
export HF_TOKEN=hf_xxxxxxxxxxxx
export HF_HUB_ENABLE_HF_TRANSFER=1
mkdir -p ~/.cache/huggingface
echo -n "$HF_TOKEN" > ~/.cache/huggingface/token

.venv/bin/python -c "from huggingface_hub import whoami; print(whoami()['name'])"
```

## Step 4 — download raw lerobot data (4 GB, source for teacher_gen)

```bash
.venv/bin/hf download \
  <your-lerobot-piper-repo> \
  --repo-type=dataset \
  --local-dir datasets/piper_insert_mouse_battery_lerobot
```

*(If already present on the box or via a different mount, skip this step and put the data at `datasets/piper_insert_mouse_battery_lerobot/`.)*


## Stage 1 — teacher_gen (regenerate ODE dataset with new teacher)

```bash
tmux new -s dd-teacher-gen
bash local_jobs/teacher_gen_720x320.sh
```

Produces `datasets/piper_720_320_warmup_regenerated_4step/` (~53 GB, 10 000 samples).
Estimated wall-clock: 8 × H100 ≈ 3–5 hours (latent grid 40×90, ~9× cheaper than 1440×640).

Detach: `Ctrl-b d`. Reattach: `tmux attach -t dd-teacher-gen`.

Verify after completion:

```bash
ls datasets/piper_720_320_warmup_regenerated_4step/{actions,images,videos,latents} | wc -l
# expected: 40000

.venv/bin/python - <<'PY'
import torch, decord
lat = torch.load("datasets/piper_720_320_warmup_regenerated_4step/latents/0.pt", map_location="cpu")
print("keys:", sorted(lat.keys()))                    # [0, 9, 18, 27, 34, 35]
print("shape:", tuple(lat[0].shape))                  # (16, 4, 40, 90)
vr = decord.VideoReader("datasets/piper_720_320_warmup_regenerated_4step/videos/0.mp4")
print("video shape:", (len(vr), *vr.get_batch([0]).asnumpy().shape[1:]))  # (13, 320, 720, 3)
PY
```

## Stage 2 — warmup training

Full run:

```bash
tmux new -s dd-warmup
NPROC=8 MAX_ITER=10000 \
  bash local_jobs/warmup_720x320.sh \
  dataloader_train.batch_size=2 trainer.save_iter=1000
```

Defaults (env vars): `NPROC=8`, `MAX_ITER=20000`, `WANDB_MODE=online`. Override with hydra args at the end.

Checkpoints land at:

```
dreamdojo_logs/cosmos_interactive/interactive_warmup/piper_720_320_no_s3_<TIMESTAMP>/checkpoints/iter_00000<n>000/
```

Estimated: 8 × 80 GB @ batch=2 ≈ 15–25 h / 10k iter (720×320 is much cheaper per-iter than 1440×640).

Dry-run first if worried about OOM:

```bash
NPROC=8 MAX_ITER=200 WANDB_MODE=offline \
  bash local_jobs/warmup_720x320.sh \
  dataloader_train.batch_size=1 trainer.save_iter=200
```

## Stage 3 — self-forcing distillation

Pick the warmup ckpt to seed the student from (usually the last or the peak-PSNR one):

```bash
WARMUP_CKPT=dreamdojo_logs/cosmos_interactive/interactive_warmup/piper_720_320_no_s3_<TIMESTAMP>/checkpoints/iter_000010000

tmux new -s dd-sf
NPROC=8 MAX_ITER=5000 \
WARMUP_CKPT="$WARMUP_CKPT" \
  bash local_jobs/self_forcing_720x320.sh
```

The teacher used for DMD distillation is the same `checkpoints/dreamdojo-piper-insert-mouse-battery-720-320-10fps-40k/model/`.

Estimated: 5000 iter × ~30 s (SF is heavier than warmup because it holds 3 nets) ≈ 40 h.

## Confirming training is running

While training:

```bash
# GPU utilization
nvidia-smi --query-gpu=index,utilization.gpu,memory.used --format=csv,noheader

# Log tail
tail -f logs/warmup_720x320_*.log        # or teacher_gen_ / self_forcing_

# Latest checkpoint
ls -t dreamdojo_logs/cosmos_interactive/interactive_warmup/*/checkpoints/ 2>/dev/null | head -3
```

Expected: all 8 GPUs at 85–100 %, log advances every 15–30 s / iter, new ckpt every `save_iter` steps.

## Notes

- `docs/QUICKSTART.md` is the 720×320 pipeline. The older 1440×640 pipeline is in `docs/QUICKSTART_1440x640.md` (if you need to reproduce that).
- All scripts read/write from repo root; run them via `bash local_jobs/xxx.sh`, not by `cd local_jobs && bash xxx.sh`.
- `dryrun` mode isn't wired into `warmup_720x320.sh` — pass `dataloader_train.batch_size=1 trainer.max_iter=200 job.wandb_mode=offline` manually as shown above.
